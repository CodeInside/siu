1. скопировать каталог gws-client-pfrf-3815 c новым именем gws-clien-{o}-{x}, где o - аббревиатура огв, x - последние 4 цифры из SID сервиса,
2. удалить из каталога .classpath, .project, .settings, target
3. переименовать каталог src/main/resources/pfrf3815 в имя клиента {o}{x}(см. п.1), и удалить содержимое,
4. скопировать wsdl и все xsd-файлы в каталог из п.3,
5. переименовать файл src/main/resources/OSGI-INF/pfrf3815.xml в имя клиента {o}{x}.xml(см. п.1), отредактировать содержимое файла указав component[name] и implementation[class], указав имя класса клиента,
6. отредактировать файл pom.xml: установить значение Export-Package как имя пакета, в котором лежит класс клиента; установить значение artifactId как название каталога проекта;
	установить значение Service-Component как путь до файла из п.5
7. создать класс клиента реализующий интерфейс ru.codeinside.gws.api.Client, пример можно посмотреть в других проектах,
8. отредактировать родительский pom.xml добавив новый созданный проект как module,
9. отредактировать uat-deployer/pom.xml добавив новый созданный проект как artifactItem,
10. выполнить команду $mvn eclipse:eclipse для pom.xml созданного проекта,
11. импортнуть новый проект в эклипс.

Процессы:
 process_request.bpmn - обработка обращения
 prepare-requests-sub.bpmn - подготовка обращения (разбор запроса с обращением в xml-файле)
 start_process_request_via_ws.bpmn - запуск процесса обработки обращения через вызов веб-сервиса(как делает МФЦ), тестовое обращение _test_documents/123.xml

Деплой процессов:
 1. через "Исполнение" -> вкладка "Управление процедурами" -> "Услуги", в "Создание услуги" задать Наименование, выбрать все Категории заявителей и подтвердить создание.
 2. вкладка "Административные процедуры" -> выбрать созданную услугу, выбрать "Новая процедура", заполнить поля, нажать "Создать",
 3. через "Выбрать файл", выбрать файл процесса и нажать "Добавить". После загрузки перевести статус добавленного процесса в "Работает".
